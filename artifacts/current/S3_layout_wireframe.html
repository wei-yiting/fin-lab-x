v4 → v5 調整

  • Tool name（yfinance_stock_quote 等）統一靠整個 tool card 的右邊（margin-left: auto）
  • 移除 header 的藍色 F logo icon，只留文字
  • 「清除對話」按鈕改為紅色系（oklch(0.7282 0.1610 27.12 / 0.08) 背景 + 紅色邊框 + 紅色文字），hover 時更亮
  • Sources 區塊移到整個 assistant 回答的最底部（在 streaming text 之後）
  • Stream Error Block 獨立為一個 assistant message block，視覺上分開

v5 → v6 調整（S3 brainstorm 決策）

  • Inline reference 改為小型 superscript 上標（.ref-sup，10px、淡藍色），點擊跳轉到 Sources 對應條目，保留 citation-to-claim 對應
  • Sources block 條目顯示 markdown reference definition 中的 title（[1]: url "title" 語法），fallback 順序：title → hostname → 截斷 URL
  • Sources block 中的 anchor 加上 id="src-N"，配合 inline superscript 跳轉
  • 依賴 backend system prompt 補強：要求 LLM 在 reference definition 中包含 title 屬性
  • 加入 .regen-btn「重新生成」ghost button，僅顯示在「最後一條 assistant message + status === 'ready'」狀態下，位置在 sources block 下方
  • Regenerate 對應 backend POST /api/v1/chat 帶 trigger: "regenerate"，backend 限制只能重生最後一個 assistant turn（_find_regenerate_target）
  • 新增 Empty State / cold start mockup：Welcome card + 4 個 suggested prompt chips（NVIDIA / Apple / Microsoft 10-Q / Google vs Amazon），UI 文案使用英文
  • Empty state 觸發條件：messages.length === 0。點擊 prompt chip → 填入 input box（不自動送出，user 可編輯後再送）
  • Empty state 時「清除對話」button disabled（沒有東西可以清）
  • 所有 hex/rgba 色值改為 oklch（perceptually uniform 色彩空間，符合 S2 shadcn 既有慣例）
  • Send button 從「vivid brand blue 填充 + glow shadow」改為「deep CTA blue 填充 + 無陰影」（Claude.ai-style 壓深的品牌色 CTA）
  • 系統色彩拆成兩階藍：--primary = oklch(0.42 0.13 255)（deep CTA, 用於 send button）；--chat-brand-accent = oklch(0.6425 0.1682 255.13)（vivid, 用於 user bubble border / source link / cursor）
